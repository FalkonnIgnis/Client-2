Network and Topology:

The best server would be client server with bus topology so that you all have access
to the same work and server. You also want the client server so that you have more security and so that no one has to put
strain on the computer to try and host something to run the game.

Software and structuring

The file system i would use is just to use many sub folders to stay
very organized. have one seperate folder for each thing inside three main ones named

  • Work Current
  • Work Upcoming
  • Archives
you will also need a web browser of somekind and to setup Linux. You might also want to check out ATOM
so you can work offline if you ever have to, plus it helps organize the code and make it easier on your eyes.

Reasoning

My reasoning behind the products I suggested are because those are the minimum required to have a good experience at the highest level of games with new games
I suggested client server so that you have security for your games and you don't need to have
someone deal with the strain of running a server to test your games. I believe the bus topology is the best because you all have
access to the same files that way especially utilising the client server as a hub. The Web browser is for Github and I also suggested Atom
so you can work offline.
